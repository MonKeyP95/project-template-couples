# Real AI suggestion cards (design)

Date: 2026-07-09
Status: approved, ready for plan

## What this is

The moss-bordered `SuggestionCard` — rendered across seven surfaces (budget,
packing, itinerary, notes, home, road, checklists) — is the **last mock AI
surface**, after chat (slice 7) and the budget drafter (2026-07-09) were made
real. Today its text comes from `src/lib/ai/suggestions.ts`: a pure
`suggestionFor(surface)` that returns one of seven hardcoded canned strings and
explicitly ignores `context` (`void context`). Its own header comment names the
seam: "make suggestionFor async and generate from the LLM client."

This slice makes it **real** — the third and final mock-to-real swap. A card's
suggestion is generated by Claude from that surface's actual trip context, so it
reflects the real trip instead of a fixed tip.

Confirmed intent (2026-07-09): "make it real (Claude)", suggest-only, and honor
the two modes (planning vs. on the road).

Scope note: this is **Piece 1 of two**. Piece 2 — a whole-trip "suggest
everything for this trip" planning button — is a separate later slice that reuses
this same seam. It is out of scope here.

## Confirmed decisions (from brainstorming)

1. **Trigger: on demand, click-to-reveal.** Cards no longer auto-generate on page
   load. Each starts collapsed showing a `/ suggest` affordance; clicking makes
   one Claude call and shows the result. No unprompted API cost, cheapest,
   simplest — and faithful to suggest-only.
2. **All 7 surfaces** are swapped.
3. **Card controls after reveal: regenerate + dismiss.** "another" re-calls Claude
   for a fresh suggestion; "dismiss" collapses back to the affordance. No "apply"
   (keeps the pure suggest-only invariant; wiring apply is a later slice).
4. **Two modes honored.** Planning surfaces suggest from planning data; the road
   surface reads the live near-horizon (today + next few days).
5. **Architecture A: one shared action + one seam** (not per-surface actions, not
   client-passed context). Mirrors exactly how chat and budget were swapped.

## Architecture

Three pieces, mirroring `sendChatMessage` -> `chatReply` and `draftBudget` ->
`draftBudgetSeeds`:

### 1. The seam — `generateSuggestion` in `src/lib/ai/claude.ts`

```
export async function generateSuggestion(
  prompt: string,
): Promise<Suggestion>
```

- Plain `anthropic.messages.create` — **no web_search** (advice from the trip
  context we already have; one fast round-trip, no discovery latency tax).
- A **forced** structured tool `propose_suggestion` with input schema
  `{ label: string, body: string }`, so the model's output shape matches today's
  `Suggestion` and `SuggestionCard` renders unchanged. Read the tool_use block
  the same way `draftBudgetSeeds` reads `propose_budget`.
- Reuses the existing `MODEL` (`claude-sonnet-4-6`). No new model constant.
- Suggest-only: returns data; never writes.
- `Suggestion` and `SurfaceKey` are imported from the renamed types module (below).

The system prompt instructs: one short, specific, actionable suggestion grounded
in the provided trip context; a terse `label` (keep the existing "/ suggested" /
"/ assistant" voice) and a one-to-two-sentence `body`; never invent facts not in
the context.

### 2. The action — `src/lib/ai/suggestion-actions.ts` (`"use server"`)

```
export async function suggestForSurface(
  surface: SurfaceKey,
  tripSlug?: string,
): Promise<{ suggestion?: Suggestion; error?: string }>
```

- Guards: `if (!(await isAiEnabled())) return { error: "AI mode is off." }`;
  resolve `getCurrentWorkspace()` (error if not signed in). Mirrors the other
  actions.
- Switches on `surface` to load context (all via **existing** queries) and build
  a plain-text context block, then calls `generateSuggestion(prompt)`:

  | surface | loads | suggestion angle |
  |---|---|---|
  | budget | `getTripBySlug` + `getBudgetItems` | budget gaps / adjustments |
  | packing | `getTripBySlug` + `getPackingItems` | missing packing items (destination + dates/season) |
  | itinerary | `getTripBySlug` + `getItineraryLocations` | itinerary gaps / ideas |
  | notes | `getTripBySlug` + `getTripNotes` | what's worth jotting |
  | road | active trip's itinerary, sliced **today + next 3 days** of events | near-horizon nudge |
  | home | `listTripsForWorkspace` (hero / soonest upcoming) | next planning step |
  | checklists | `listChecklists` | a template or gap idea |

- **Trip surfaces** (budget/packing/itinerary/notes) require `tripSlug`; a missing
  one returns a soft error. **road** takes `tripSlug` (the on-the-road page knows
  `trip.slug`) and slices the loaded itinerary days to today + the next 3.
  **home/checklists** take no slug and resolve their own workspace-level context.
- Each branch builds a compact prompt: trip name/dates/destination where
  relevant + the surface's data as short lines. Keep prompts small.

### 3. The client — `src/components/ai-suggestion.tsx`

- Gains a `tripSlug?` prop (passed by the 5 trip surfaces + road; omitted by
  home/checklists).
- Still `useAiMode()`-gated: renders nothing when AI is off.
- State machine: **collapsed** (a `/ suggest` affordance) -> click -> **loading**
  ("...thinking") -> **revealed** (the `SuggestionCard` with the suggestion, an
  "another" control that re-calls, and "dismiss" that returns to collapsed).
- On error, show a small clay note (e.g. "couldn't reach the assistant") and
  return to the affordance, matching the budget drafter's failure treatment.
- `SuggestionCard` (in `@/components/together`) is unchanged.

### 4. Types — rename `suggestions.ts` -> `suggestion-types.ts`

- Move `SurfaceKey` and `Suggestion` into `src/lib/ai/suggestion-types.ts` (pure,
  no server-only import) so the client, the action, and the seam all import the
  types from one client-safe module (the `*-types.ts` split convention).
- **Delete** the canned `SUGGESTIONS` record and `suggestionFor` — nothing reads
  them after the swap.
- Update the 7 call sites' `AiSuggestion surface="..."` imports if the type import
  path changes (the component owns the import; call sites pass a string literal).

## Call sites (7, one prop each)

The 5 trip-tab surfaces are under `/trips/[slug]`, so they already have the slug;
`on-the-road/page.tsx` has `trip.slug`. Home and checklists pass none.

| file | change |
|---|---|
| `budget-tab.tsx` | `surface="budget" tripSlug={slug}` |
| `packing-tab.tsx` | `surface="packing" tripSlug={slug}` |
| `itinerary-tab.tsx` | `surface="itinerary" tripSlug={slug}` |
| `notes-tab.tsx` | `surface="notes" tripSlug={slug}` |
| `on-the-road/page.tsx` | `surface="road" tripSlug={trip.slug}` |
| `home/page.tsx` | `surface="home"` (no slug) |
| `checklists/page.tsx` | `surface="checklists"` (no slug) |

(Each trip tab already receives the trip; the slug is the route param — thread it
to the card. Confirm each tab has the slug in scope during the plan.)

## Invariants honored

- **AI-gated**: the action self-gates on `isAiEnabled()`; the card self-gates on
  `useAiMode()`. When off, no card and no call.
- **Suggest-only**: nothing under `lib/ai` writes; the card only displays.
- **One seam**: all Claude calls stay in `claude.ts`; the action is the caller.
- **No migration, no deps, no new vendor.**
- **Two modes**: planning surfaces read planning data; road reads the live
  near-horizon.

## Deferred

- A working "apply" (per-surface writes) — breaks suggest-only; separate slice.
- Persisted / shared dismissal (currently session/collapse-local).
- Streaming.
- The whole-trip "suggest everything for this trip" planning button (Piece 2).
- Caching a generated suggestion (each click is a fresh call; fine at on-demand
  volume).
